Mission Dashboard

Left Status Pane. Contains status information about the teams which are interacting with the system. A team appears in the pane when they are either in-air (TakeoffOrLandingEvent) or active (recent interop requests).

Right 3D Display. The display pulls data (mission config, telemetry, obstacles) from the backend at a regular rate and displays it. The display is initially centered at the MissionConfig's home position (green sphere). Additional elements are mission elements (white spheres, e.g. SRIC), waypoints (blue spheres), search grid (light blue spheres), obstacles (red spheres and cylinders), and UAS positions (yellow spheres).

System Menu

Live View (KML). Downloads a KML file to view live mission and telemetry data in Google Earth.

Export Data (KML). Exports mission and telemetry data as a KML file which can be viewed in Google Earth.

Edit Data. Opens the Django admin page for editing raw data, and currently is the primary mechanism for configuring the server.

Clear Cache. Forces the caches to be cleared, so updated are viewed instantly.

Mission Menu

Mission Dashboard. Opens this page.

Review Odlc. Interface to review object detection imagery, review of which must be done prior to team evaluation.

Evaluate Teams (CSV). Evaluates the teams on the mission and saves a CSV file.
